📋 PROJECT SUMMARY
═══════════════════════════════════════════════════════════════════════════

MedProof is a production-ready decentralized medical records platform that
enables patients to securely upload encrypted medical records and allows
doctors to verify them on-chain with immutable audit trails.

✅ STATUS: COMPLETE & READY FOR TESTNET DEPLOYMENT


🎯 KEY FEATURES
═══════════════════════════════════════════════════════════════════════════

Patient Features:
  ✅ Connect Freighter wallet
  ✅ Upload medical records (PDF, images)
  ✅ Client-side AES-256 encryption
  ✅ View verification status
  ✅ Track record history

Doctor Features:
  ✅ Connect Freighter wallet
  ✅ Review pending records
  ✅ Approve or reject with reasons
  ✅ On-chain verification decisions
  ✅ Activity audit trail

Public Features:
  ✅ Verify records without authentication
  ✅ Query by record hash (IPFS CID)

  ✅ Transparent audit trail

🔄 USER FLOWS
═══════════════════════════════════════════════════════════════════════════

Patient Upload Flow:
  1. Connect Freighter wallet
  2. Select role: "Patient"
  3. Upload medical record file
  4. File encrypted client-side (AES-256)
  5. File uploaded to IPFS → CID returned
  6. Metadata uploaded to IPFS
  7. CID stored on blockchain
  8. Transaction signed via Freighter
  9. Record appears with "pending" status

Doctor Verification Flow:
  1. Connect Freighter wallet
  2. Select role: "Doctor"
  3. View pending records in queue
  4. Review record details
  5. Click "Approve" or "Reject"
  6. Enter reason (if rejecting)
  7. Transaction signed via Freighter
  8. Verification stored on blockchain
  9. Status updated to "verified" or "rejected"

Public Verification Flow:
  1. Visit /verify page (no wallet needed)
  2. Enter record hash (IPFS CID)
  3. Query blockchain for verification status
  4. Display verification details
